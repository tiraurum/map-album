完整架构设计

技术栈、组件树、数据模型、交互流程

技术栈

框架React 18 + Vite
地图Leaflet + react-leaflet
地图瓦片CartoDB Dark Matter（暗色风格）
存储IndexedDB (Dexie.js) 存照片+数据
样式CSS Modules / 内联暗色主题
城市数据~300地级市坐标 JSON（内嵌）

组件树

App
├── TopBar              ← 标题 + "已探索 X/300 城" 统计
├── MapView             ← 中国地图区域 (Leaflet)
│   ├── TileLayer       ← 暗色地图瓦片
│   ├── CityMarkers     ← 所有城市标记点
│   │   ├── VisitedMarker   ← 去过城市 (红色发光)
│   │   └── UnvisitedMarker ← 未去过 (灰色小点)
│   ├── RouteLines      ← 去过城市之间的连线
│   └── CityPopup       ← 点击城市弹出的信息浮窗
├── Sidebar             ← 右侧边栏: 去过城市列表
│   └── CityCard[]      ← 每个去过城市 (缩略图+名称+统计)
└── DetailPanel         ← 底部/弹窗: 城市详情/编辑
    ├── PhotoGrid       ← 照片墙 (上传/展示)
    └── DescriptionEditor ← 文字描述编辑

数据模型

// 城市数据 (内嵌 JSON)
City {
  id: number           // 城市编码
  name: string         // 城市名
  province: string     // 所属省份
  lat: number          // 纬度
  lng: number          // 经度
}

// 用户存储数据 (IndexedDB)
CityRecord {
  cityId: number       // 关联城市 ID
  visited: boolean     // 是否去过
  photos: Photo[]      // 照片列表 (base64)
  description: string  // 文字描述
  visitDate: string    // 到访时间
  createdAt: string    // 创建时间
  updatedAt: string    // 更新时间
}

Photo {
  id: string           // 唯一 ID
  dataUrl: string      // base64 图片数据
  caption: string      // 图片说明 (可选)
  createdAt: string    // 上传时间
}

交互流程

操作 行为
页面加载 加载 IndexedDB 数据 → 合并城市标记 → 渲染地图
点击未去过城市 弹窗显示城市名 + "标记为去过" 按钮
标记为去过 → 保存到 IndexedDB → 标记变红色发光 → 侧边栏新增
点击去过城市 显示详情面板（照片+描述），可编辑
上传照片 文件选择 → 压缩 → base64 → IndexedDB → 更新显示
编辑描述 保存到 IndexedDB → 下次加载自动恢复
